The 3x3x3 RED LED CUBE

The first one I made was just a test of what the Arduino could do, nothing more. It was a very simple 3x3x3 RED LED cube, controlled by the Arduino. It used pulse width modulation (PWM) to control the brightness of each individual LED, and was a fairly simple project. But it did teach me the fundamentals of making such projects, such as using a transistor for each cathode wire to ground the LEDs to make them light up. It also used very basic algorithms to create simple patterns with the 27 LEDs. It was a fairly intuitive project, but nonetheless it was still a very fun project to make.

The 4x4x4 BLUE LED CUBE

After creating the 3x3x3 RED LED cube, I wanted to create something bigger, something more impressive. So I took the next natural step and created the 4x4x4 cube, which was more difficult but also more rewarding at the same time.

Creating this naturally came with it's own share of challenges. For example:

  • It became too expensive to devote an entire Arduino to each individual project. It was okay for experimental purposes but it became impossible to use a $35 dollar board for each of my projects. So I decided to replace the Arduino in my projects with it’s microprocessor, the ATMEGA328, which at it’s core is what the Arduino really is. This brought down the cost of each project to a manageable $5.
  • Then the challenge changed to how I would control each LED, and it’s respective brightness, since I no longer had enough PWM capable pins to support pulse width modulation to each LED. (The 3x3x3 already required 9 PWM pins to control) The solution turned out to be completely changing the control structure of the matrix. Since PWM was no long viable, another technique had to be used, and that was bit angle modulation (BAM). BAM allowed me to control each of the 64 LEDs and give each a 16-bit brightness level. This allowed me to use all 13 digital pins on the ATMEGA328, instead of limiting myself to the scarce number of PWM pins it had.

But after finishing the project, I learned a great deal of things, for example:

  • How to control a large matrix of LEDs with only a limited number of digital outputs and an even more limited number of PWM outputs.
  • Various software techniques which I used to program the animations you see in the video of the cube.
  • How to make a finished project look visually appealing and presentable.

This project was lots of fun, but I found that it wasn't enough. I wanted more colours, more animations, the 4x4x4 BLUE cube just wasn’t enough. So I then took the next step forward, and created the 4x4x4 RGB LED CUBE, which was much, much more sophisticated then anything I've ever built before.

4x4x4 RGB LED CUBE - The Finale

The 4x4x4 RGB LED CUBE was the final project in this series I decided to make, simply because anything larger would just consume too much time to complete, so I decided to utilize all my knowledge to build something cool to end it off. As mentioned, this was something that was much more sophisticated than anything I’ve done before, but it was fun to face the challenge.

Designing the cube presented me with many problems, for example:

  • Finding a way to control 192 LEDs individually.
  • Giving each LED a 16-bit brightness level to create any colour.
  • Sourcing the power to all 192 LEDs.
  • Finding a way to control 192 LEDs with only 13 digital output pins from the ATMEGA328 microprocessor.

I solved these problems with a new technique I discovered when designing the cube, which was using shift registers. Shift registers was the major difference between the RGB cube and the BLUE cube, simply because there were too many LEDs in the RGB cube. There was no possibility of controlling 192 LEDs without additional help. Shift registers allowed me to control all 192 LEDs with only 3 digital output pins from the ATMEGA328, and that is something I am proud of. Using this technique I was able to power all 192 LEDs and it allowed them each to have a 16-bit brightness level.

This project was definitely the most fun I’ve had building anything, simply because of the complexity that was involved. Soldering over 500 joints and painstakingly checking each was tough but in the end worth it because of all that I’ve learned from this project. It was truly a masterpiece.
